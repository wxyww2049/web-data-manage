        "content": "# 1.new和malloc的区别（使用和原理）⭐\n\n## **new的定义：**\n\nnew是C++的关键字，用于动态分配内存并创建对象。它可以根据类型自动计算所需内存空间，并调用对象的构造函数进行初始化。在使用new分配内存后，需要使用delete来释放这些内存空间，以防止内存泄漏。\n\n## **malloc的定义：**\n\nmalloc是C语言的库函数，用于动态分配一块指定大小的内存块，并返回其地址。需要注意的是，使用malloc分配内存后，需要使用free来释放这些内存空间，以防止内存泄漏。\n\n## **new与malloc的区别：(简单理解)**\n\n**语法：new是C++的关键字，而malloc是C语言的库函数。**\n\n**类型安全：new操作符会根据类型自动计算所需内存大小，并进行类型匹配，返回的是对象类型指针；而malloc需要手动计算内存大小，并使用强制类型转换，返回的是void指针。**\n\n构造函数与析构函数的调用：new会自动调用对象的构造函数进行初始化，而malloc不会调用构造函数，得到的内存空间内容是未初始化的。\n\n**内存泄漏的检测：new可以通过异常机制检测内存分配失败，而malloc在分配失败时返回NULL，需要手动检查。**\n\n重载和自定义类型：new操作符可以重载，并能够与自定义类型的构造函数和析构函数配合使用；而malloc是库函数，不会调用自定义类型的构造和析构函数\n\n## **new与malloc使用区别**\n\n    \n    \n    #include <iostream>\n    #include <cstdlib>\n    \n    int main() {\n        // 使用new进行动态内存分配和释放\n        int* newPtr = new int(10);\n        std::cout << \"Value allocated with new: \" << *newPtr << std::endl;\n        delete newPtr;\n    \n        // 使用malloc进行内存分配和释放\n        int* mallocPtr = (int*)malloc(sizeof(int));\n        if (mallocPtr != nullptr) {\n            *mallocPtr = 20;\n            std::cout << \"Value allocated with malloc: \" << *mallocPtr << std::endl;\n            free(mallocPtr);\n        }\n    \n        return 0;\n    }\n    \n\n# 2.struct和class的区别⭐\n\n  1. **默认访问权限** ：struct中的成员默认为公共(public)，而class中的成员默认为私有(private)。\n  2. **默认继承方式** ：struct中的继承方式默认为公共(public)，class中的继承方式默认为私有(private)。\n  3. **使用习惯** ：struct适合用于简单的数据结构，class适合用于复杂的数据类型和实现面向对象编程。\n  4. **成员变量和成员函数** ：struct中的成员变量和成员函数默认为公共，而class中的成员变量和成员函数默认为私有。\n  5. **访问控制** ：struct中的成员在外部可直接访问，而class中的成员需要使用公共的成员函数来访问。\n  6. **默认的构造函数和析构函数** ：class中会自动生成默认的构造函数和析构函数，而struct中不会。\n\n# **3.char和int之间的转换**\n\n  1. 将char转换为int：可以直接将char类型的变量赋值给int类型的变量，将字符对应的ASCII码值赋给int变量。\n\n    \n    \n    char c = 'A';\n    int i = c; // 未初始化的指针，成为野指针\n    \n        // 使用野指针会导致未定义的行为\n        *ptr = 5; // 解引用野指针，可能导致程序崩溃\n    \n        return 0;\n    }\n    \n\n**悬挂指针（Dangling\nReference）：指针最初指向的内存已经被释放了的一种指针。指针指向的内存已释放，但指针的值没有被清零，对悬空指针操作的结果不可预知**\n\n示例代码：\n\n    \n    \n    int* createInt() {\n        int value = 5;\n        int* ptr = &value;\n        return ptr; // 返回指向局部变量的指针\n    }\n    \n    int main() {\n        int* danglingPtr = createInt(); // 指向已释放的内存\n    \n        // 对悬挂指针操作的结果不可预知\n        int value = *danglingPtr; // 解引用悬挂指针，可能导致未定义的行为\n    \n        return 0;\n    }\n    \n\n# 5.NULL和nullptr区别⭐\n\n类型不同：NULL是宏定义或整数值0，而nullptr是C++11引入的关键字，表示空指针。\n\n安全性不同：NULL可能导致函数调用二义性问题，nullptr更安全，不会被错误解释为整型。\n\n上下文匹配不同：NULL可以用于整型类型的上下文，nullptr只能用于指针类型的上下文。\n\n# 6.指针常量和常量指针有何区别⭐\n\n**指针常量** （Pointer to a\nconstant）：指针常量是指指针本身是不可更改的，即指针变量自身的值不能被修改，但指向的值可以修改。声明指针常量时，需要在指针类型前加上 `const`\n修饰符。例如：\n\n    \n    \n    int* const ptr;  // ptr 是指向常量的指针，指向的值不能通过 ptr 修改，但可以通过其它方式修改\n    \n    \n\n在这个例子中，`ptr` 是一个指向 `int` 类型常量的指针，它的值不能被修改，但可以通过其他途径修改指向的值。\n\n**常量指针** （Constant\npointer）：常量指针是指指针指向的地址不能更改，即指针变量指向的地址不能被修改，但可以通过指针修改指向的值。常量指针在声明时需要使用 `const`\n修饰符将指针指向的类型声明为常量。例如：\n\n    \n    \n    int x = 5;\n     const int * ptr = &x;  // ptr 是常量指针，指向的地址不能通过 ptr 修改，但可以通过 *ptr 修改值\n    \n    \n    \n    \n      \n    \n    \n    \n    ptr是常量指针，不可以修改ptr指向地址的值，可以修改ptr 指向的地址。  \n    \n\n总结：\n\n  * 指针常量表示指针本身的值不能修改，但可以通过指针修改指向的值。\n  * 常量指针表示指针指向的地址不能修改，但可以通过指针修改指向地址的值\n\n# **7.虚拟内存和物理内存的区别** ⭐\n\n**虚拟内存（Virtual Memory）和物理内存（Physical Memory）是计算机系统中存储和管理数据的两个概念：**\n\n  * 物理内存是计算机中的实际硬件内存，由RAM芯片组成。\n  * 虚拟内存是对物理内存的扩展，使用磁盘空间来模拟更大的内存容量。\n\n**它们之间的区别包括：**\n\n  * 大小：物理内存的容量是固定的，而虚拟内存的大小可以超过物理内存的容量。\n  * 访问速度：物理内存的访问速度较快，而虚拟内存的访问速度相对较慢，因为它需要与磁盘进行交互。\n  * 地址空间：物理内存使用物理地址进行访问，而虚拟内存使用虚拟地址，通过内存管理单元（MMU）映射到物理内存。\n  * 管理方式：物理内存的管理相对简单，而虚拟内存的管理涉及页表和页面置换等技术。\n  * 可用空间：物理内存的可用空间有限，而虚拟内存可以提供更大的可用空间，因为它可以使用磁盘空间作为扩展。\n\n总之，虚拟内存扩展了物理内存的容量，并提供了更灵活的内存管理机制，但付出的代价是访问速度较慢。\n\n# 8.重载、重写和隐藏的区别⭐\n\n**重载（Overloading）：**\n\n  * 重载是在同一个作用域内定义多个相同名称但参数列表不同的函数或方法。\n  * 重载函数可以根据不同的参数数量或类型来执行不同的操作。\n  * 重载通过函数名和参数列表来区分不同的函数。\n\n举例代码：\n\n    \n    \n    #include <iostream>\n    \n    void printNumber(int num) {\n        std::cout << \"Integer number: \" << num << std::endl;\n    }\n    \n    void printNumber(double num) {\n        std::cout << \"Floating-point number: \" << num << std::endl;\n    }\n    \n    int main() {\n        printNumber(10);\n        printNumber(3.14);\n        return 0;\n    }\n    \n    \n\n**重写（Override）：**\n\n  * 重写是指子类重新定义从父类继承的虚函数，使其具有不同的实现。\n  * 重写的函数签名（函数名、参数列表和返回类型）必须与被重写函数相同。\n  * 在运行时，根据具体的对象类型，调用的是子类重写的版本。\n\n举例代码：\n\n    \n    \n    #include <iostream>\n    \n    class Base {\n    public:\n        virtual void sayHello() {\n            std::cout << \"Hello from Base class!\" << std::endl;\n        }\n    };\n    \n    class Derived : public Base {\n    public:\n        void sayHello() override {  // 使用 override 关键字表明重写了父类的函数\n            std::cout << \"Hello from Derived class!\" << std::endl;\n        }\n    };\n    \n    int main() {\n        Base* basePtr = new Derived();\n        basePtr->sayHello();  // Output: \"Hello from Derived class!\"\n        delete basePtr;\n        return 0;\n    }\n    \n    \n\n**隐藏（Hiding）：**\n\n  * 隐藏是指在派生类中定义与父类具有相同名称的成员函数，使其隐藏父类中的同名函数。\n  * 隐藏函数与父类的函数没有多态性，只有通过对象的实际类型调用时才会调用相应的函数。\n\n举例代码：\n\n    \n    \n    #include <iostream>\n    \n    class Base {\n    public:\n        void sayHello() {\n            std::cout << \"Hello from Base class!\" << std::endl;\n        }\n    };\n    \n    class Derived : public Base {\n    public:\n        void sayHello() {\n            std::cout << \"Hello from Derived class!\" << std::endl;\n        }\n    };\n    \n    int main() {\n        Base baseObj;\n        Derived derivedObj;\n        \n        baseObj.sayHello();    // Output: \"Hello from Base class!\"\n        derivedObj.sayHello(); // Output: \"Hello from Derived class!\"\n        \n        Base* basePtr = new Derived();\n        basePtr->sayHello();   // Output: \"Hello from Base class!\"\n        \n        delete basePtr;\n        return 0;\n    }\n    \n\n# 9.简述面向对象的三大特性\n\n**面向对象编程（OOP）的三大特性是封装、继承和多态。下面对每个特性进行简要说明：**\n\n**封装（Encapsulation）：**\n\n  * 封装是将数据和操作封装在一个单元（类）中的机制。\n  * 通过封装，实现类的成员变量和成员函数作为一个整体进行管理和操作。\n  * 封装隐藏了数据的具体实现细节，只暴露出必要的接口，提供了更好的安全性和可维护性。\n  * 通过访问修饰符（公有、私有、保护），控制对类的成员的访问权限。\n\n**继承（Inheritance）：**\n\n  * 继承是通过创建派生类来扩展和重用已有类的机制。\n  * 基类（父类）是已经定义的类，派生类（子类）继承了基类的属性和方法。\n  * 子类可以自定义新的属性和方法，也可以覆盖或扩展继承的父类的属性和方法。\n  * 继承实现了类与类之间的关系，实现了代码的重用和扩展。\n\n**多态（Polymorphism）：**\n\n  * 多态是指同一个接口可以由不同的对象以不同的方式进行实现和响应的能力。\n  * 多态允许使用基类的指针或引用来引用派生类的对象，实现了多种形态的使用。\n  * 编译时多态使用函数重载和运算符重载；运行时多态通过虚函数实现（动态绑定）。\n  * 多态提高了代码的灵活性和可扩展性，使得程序更具有可读性和可维护性。\n\n# 10.什么是多态？\n\n利用虚函数，基类指针指向基类对象时就使用基类的成员（包括成员函数和成员变量），指向派生类对象时就使用派生类的成员。\n基类指针可以按照基类的方式来做事，也可以按照派生类的方式来做事，它有多种形态，或者说有多种表现方式，我们将这种现象称为多态（Polymorphism）。\n\n**代码举例：**\n\n    \n    \n    #include <iostream>\n    \n    class Base {\n    public:\n        virtual void print() {\n            std::cout << \"This is the Base class\" << std::endl;\n        }\n    };\n    \n    class Derived : public Base {\n    public:\n        void print() override {\n            std::cout << \"This is the Derived class\" << std::endl;\n        }\n    };\n    \n    int main() {\n        Base* basePtr;\n        \n        Base baseObj;\n        Derived derivedObj;\n        \n        basePtr = &baseObj;\n        basePtr->print(); 客户端收到服务器的响应后，再发送一个ACK（确认）响应，表示已经收到服务器的确认，连接建立完成。\n\n四次挥手是指在断开TCP连接时，客户端和服务器之间进行的一系列挥手动作。具体步骤如下：\n\n  1. 客户端向服务器发送一个FIN（结束）请求，表示不再发送数据。\n  2. 服务器收到请求后，向客户端发送一个ACK响应，表示已经收到客户端的请求。\n  3. 服务器再发送一个FIN请求，表示服务器也不再发送数据。\n  4. 客户端收到服务器的请求后，向服务器发送一个ACK响应，表示已经收到服务器的请求，连接断开完成。\n\n通过三次握手和四次挥手，客户端和服务器可以建立和断开连接，并保证数据的可靠传输。\n\n### 4\\. 缓存穿透和雪崩的原因和解决方法\n\n缓存穿透和雪崩是常见的缓存问题，它们的原因和解决方法如下：\n\n  1. 布隆过滤器你了解吗\n\n**布隆过滤器**\n是一种用于判断一个元素是否存在于集合中的数据结构，它通过使用多个哈希函数和位数组来实现。具体来说，布隆过滤器会将每个元素通过多个哈希函数映射到位数组中的多个位置，将对应位置的位设置为1。当要判断一个元素是否存在时，布隆过滤器会对该元素进行相同的哈希映射，然后查看对应位置的位是否都为1，若都为1，则认为元素存在于集合中，若存在任一位为0，则认为元素不存在于集合中。\n\n**布隆过滤器**\n的主要作用是在大规模数据集中快速判断一个元素是否存在，常用于缓存系统、网络爬虫、垃圾邮件过滤等场景，可以有效地减少对底层存储系统的查询压力。\n\n布隆过滤器的**优点** 包括：\n\n  1. 空间效率高：布隆过滤器只需要使用一个位数组和多个哈希函数来表示集合，相比使用传统的哈希表或者树等数据结构，布隆过滤器的空间占用更小。\n  2. 查询效率高：布隆过滤器通过多个哈希函数将元素映射到多个位置，所以查询一个元素只需要进行几次位操作，时间复杂度较低。\n  3. 可扩展性好：布隆过滤器支持动态添加元素，可以根据需要进行扩展。\n\n布隆过滤器的**缺点** 包括：\n\n  1. 存在一定的误判率：由于多个元素可能映射到同一个位，所以当判断一个元素是否存在时，存在一定的误判率，即可能将不存在的元素误判为存在。\n  2. 不支持元素的删除：由于多个元素可能映射到同一个位，所以无法准确地删除一个元素，只能通过重新构建布隆过滤器来实现。\n\n> 总的来说，布隆过滤器是一种高效的数据结构，适用于对查询效率要求较高、对误判率能够容忍的场景。\n\n###  1. 创建对象A时，首先会检查A是否在一级缓存中，如果在则直接返回A的实例。\n  2. 如果A不在一级缓存中，则会创建一个A的早期引用，并将其放入二级缓存中。\n  3. 接着，Spring会开始创建A对象的实例，并将其放入一级缓存中。\n  4. 在创建A对象的过程中，如果发现A依赖于B，那么Spring会先去创建B对象。\n  5. 创建B对象时，同样会检查B是否在一级缓存中，如果在则直接返回B的实例。\n  6. 如果B不在一级缓存中，则会创建一个B的早期引用，并将其放入二级缓存中。\n  7. 接着，Spring会开始创建B对象的实例，并将其放入一级缓存中。\n  8. 在创建B对象的过程中，如果发现B又依赖于A，那么Spring会从二级缓存中获取A的早期引用，并注入到B中。\n  9. 当A和B的创建过程都完成后，Spring会将A和B的实例放入一级缓存中，并将二级缓存中的早期引用清除。\n  10. 最后，Spring会将A和B的实例注入到彼此的属性中，完成循环依赖的解决。\n\n通过使用三级缓存，Spring能够在对象创建过程中解决循环依赖的问题，保证对象的正确创建和注入。这种机制使得Spring能够处理复杂的依赖关系，提高了应用程序的灵活性和可维护性。\n\n### 13\\. mysql中有哪些锁\n\n在MySQL中，常见的锁包括以下几种：\n\n  1. 共享锁（Shared Lock）：也称为读锁，多个事务可以同时持有共享锁，用于保证读操作的一致性。共享锁之间不会互相阻塞，但会与排它锁互斥。\n  2. 排它锁（Exclusive Lock）：也称为写锁，只有一个事务可以持有排它锁，用于保证写操作的原子性。排它锁与其他任何锁都互斥，包括共享锁和排它锁。\n  3. 记录锁（Record Lock）：用于保护单个记录的锁，可以是共享锁或排它锁。记录锁是在存储引擎层实现的，不同的存储引擎可能有不同的实现方式。\n  4. 表锁（Table Lock）：锁定整个表，可以是共享锁或排它锁。表锁是在MySQL服务器层实现的，对整个表进行锁定，会对其他事务的读写操作产生阻塞。\n  5. 行锁（Row Lock）：也称为行级锁，用于保护表中的行数据。行锁可以是共享锁或排它锁，不同的事务可以同时持有不同行的共享锁，但只能有一个事务持有某一行的排它锁。\n\n需要注意的是，MySQL的锁机制是基于存储引擎实现的，不同的存储引擎可能有不同的锁实现方式。常见的存储引擎包括InnoDB、MyISAM等，它们在锁的粒度、并发性能等方面有所不同。\n\n### 14\\. 锁的是什么东西\n\n在MySQL中，锁是用于控制并发访问的机制，用于保证数据的一致性和完整性。锁可以应用在不同的粒度上，包括表级锁和行级锁。\n\n表级锁是对整个表进行加锁，当一个事务获取了表级锁后，其他事务无法对该表进行修改操作，只能进行读取操作。表级锁的优点是简单、粗粒度，但是并发性较差。\n\n行级锁是对表中的行进行加锁，当一个事务获取了某一行的锁后，其他事务可以继续对其他行进行操作，只有对同一行的操作会被阻塞。行级锁的优点是并发性好，但是实现相对复杂。\n\nMySQL中的锁可以分为共享锁（读锁）和排他锁（写锁）。共享锁可以被多个事务同时获取，用于读取操作，不会阻塞其他事务的读取操作。排他锁只能被一个事务获取，用于写入操作，会阻塞其他事务的读取和写入操作。\n\n锁的使用需要根据具体的业务场景和并发访问需求进行合理的设计和配置，以保证数据的一致性和并发性。\n\n### 15\\. 线程和进程的区别\n\n线程和进程是操作系统中的两个重要概念，它们有以下区别：\n\n  1. 定义：进程是程序的执行实例，是操作系统进行资源分配和调度的基本单位；线程是进程中的一个执行单元，是操作系统进行调度和执行的基本单位。\n  2. 资源占用：进程拥有独立的地址空间、文件描述符、堆栈等资源，相互之间不共享；线程与所属进程共享地址空间和其他资源，线程之间可以直接访问同一进程的数据。\n  3. 切换开销：进程切换需要保存和恢复整个进程的上下文，开销较大；线程切换只需要保存和恢复线程的上下文，开销较小。\n  4. 通信和同步：进程之间通信需要使用进程间通信（IPC）机制，如管道、消息队列、共享内存等；线程之间通信可以直接读写共享变量，同步可以使用互斥锁、条件变量等机制。\n  5. 独立性：进程是独立的执行实体，拥有自己的执行状态和控制流；线程是进程的一部分，共享进程的资源，执行状态和控制流与所属进程相互依赖。\n\n总结来说，进程是资源分配的基本单位，线程是执行调度的基本单位。进程之间相互独立，线程之间共享资源。进程切换开销大，线程切换开销小。进程间通信需要使用IPC机制，线程间通信可以直接读写共享变量。\n\n### 16\\. 线程的几种状态\n\n在Java中，线程有以下几种状态：\n\n  1. 新建（New）：当线程对象被创建时，它处于新建状态。此时线程还没有开始执行。\n  2. 运行（Runnable）：当调用线程的start()方法后，线程进入运行状态。此时线程正在执行任务。\n  3. 阻塞（Blocked）：当线程被阻塞时，它暂时停止执行。线程可能会进入阻塞状态的原因有多种，比如等待某个资源、等待输入/输出等。\n  4. 等待（Waiting）：线程进入等待状态是因为调用了Object类的wait()方法、Thread类的join()方法或LockSupport类的park()方法。在等待状态下，线程会释放持有的锁。\n  5. 超时等待（Timed Waiting）：线程进入超时等待状态是因为调用了Thread类的sleep()方法、Object类的wait(long timeout)方法、Thread类的join(long millis)方法或LockSupport类的parkNanos()方法。\n  6. 终止（Terminated）：线程执行完任务后，或者因异常而结束，线程进入终止状态。\n\n以上就是Java中线程的几种状态。\n\n### 17\\. 怎么样能使线程进入阻塞状态\n\n可以使用以下方法使线程进入阻塞状态：\n\n  1. 使用Object类的wait()方法：调用wait()方法可以使当前线程进入阻塞状态，并释放对象的锁。其他线程可以通过调用相同对象的notify()或notifyAll()方法来唤醒被阻塞的线程。\n  3. 使用Thread类的join()方法：调用join()方法可以使当前线程等待另一个线程执行完毕后再继续执行。例如，thread.join()将使当前线程阻塞，直到thread线程执行完毕。\n  4. 使用Lock类的lock()方法：通过使用Lock类的lock()方法获取锁，可以使线程进入阻塞状态，直到获取到锁为止。例如，lock.lock()将使当前线程阻塞，直到获取到锁。\n\n以上是几种常见的使线程进入阻塞状态的方法。根据具体的需求和场景，选择适合的方法来实现线程的阻塞。\n\n### 18\\. wait（）和sleep（）的区别\n\n`wait()`和`sleep()`是两个不同的方法，`wait()`方法是Object中的，`sleep()`是Thread中的。都是用于控制程序的执行。\n\n`wait()`方法是用于线程间通信的，它会使当前线程进入等待状态，直到子线程结束或收到指定的信号。在等待期间，当前线程会被挂起，不会占用CPU资源。\n\n`sleep()`函数是用于线程间的延时操作，它会使当前线程进入睡眠状态，暂停执行一段指定的时间。在睡眠期间，当前线程会被挂起，不会占用CPU资源。\n\n总结来说，`wait()`是用于线程间通信，而`sleep()`是用于线程间的延时操作。\n\n### 20\\. 类的加载的过程\n\nJVM中类的加载过程可以分为以下几个步骤：\n\n  1. 加载（Loading）：将类的字节码文件加载到内存中。这个过程可以通过类加载器（ClassLoader）来完成。类加载器会根据类的全限定名（包括包名和类名）来查找并加载对应的字节码文件。\n  2. 验证（Verification）：验证字节码文件的正确性和安全性。这个过程会检查字节码文件的结构是否符合规范，并且会进行一些静态分析，以确保字节码文件不会引发安全问题。\n  3. 准备（Preparation）：为类的静态变量分配内存，并设置默认初始值。在这个阶段，JVM会为类的静态变量分配内存空间，并根据变量的类型设置默认的初始值（例如，int类型的变量默认为0）。\n  4. 解析（Resolution）：将符号引用解析为直接引用。在Java中，类之间的引用是通过符号引用来表示的，而在解析阶段，JVM会将这些符号引用解析为直接引用，以便后续的访问和调用。\n  5. 初始化（Initialization）：执行类的初始化代码。在这个阶段，JVM会执行类的静态初始化代码，包括静态变量的赋值和静态代码块的执行。类的初始化是在类第一次被使用时触发的，例如创建类的实例、访问类的静态变量或调用类的静态方法。\n  6. 使用（Usage）：使用类的实例或调用类的方法。在类被初始化之后，就可以使用类的实例或调用类的方法了。\n  7. 卸载（Unloading）：卸载不再使用的类。当一个类不再被引用，并且没有任何活动的实例时，JVM会卸载该类，释放相关的内存空间。\n\n以上就是JVM中类的加载过程的主要步骤。\n\n### 21\\. 说一说双亲委派机制\n\nJVM中的双亲委派机制是一种类加载机制，它的目的是保证Java类的安全性和一致性。在双亲委派机制下，当一个类加载器收到加载类的请求时，它首先会将这个请求委派给它的父类加载器去完成，只有当父类加载器无法完成加载时，才会由当前类加载器自己去加载。\n\n这种机制的好处在于可以避免类的重复加载，保证类的唯一性。当一个类加载器收到加载类的请求时，它会首先向上委派给父类加载器，如果父类加载器能够找到并加载这个类，那么就直接返回这个类的Class对象；如果父类加载器无法找到这个类，那么子类加载器才会尝试自己去加载。\n\n双亲委派机制的具体实现是通过ClassLoader类的loadClass()方法来实现的。在loadClass()方法中，首先会检查是否已经加载过该类，如果已经加载过，则直接返回已加载的Class对象；如果没有加载过，则会调用父类加载器的loadClass()方法来尝试加载，如果父类加载器加载失败，则会调用自己的findClass()方法来加载类。\n\n总结来说，双亲委派机制保证了类的加载顺序，从而保证了类的唯一性和一致性。它的核心思想是\"上级加载器优先\"，即优先使用父类加载器来加载类，只有在父类加载器无法加载时才由子类加载器来加载。这种机制有效地避免了类的重复加载和冲突，提高了系统的安全性和稳定性。\n\n###
